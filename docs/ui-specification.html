0. デザイン原則・トーン

  • 業務効率最優先: 日々の作業時間を短くする。装飾より情報密度。
  • ステップを明示: 1.アップロード → 2.処理中 → 3.結果 の3ステップを常にヘッダに表示。
  • ブロックしない: 処理中も画面で進行状況が見える（固まったと誤解させない）。
  • 失敗を隠さない: 未マッチ品番と理由は同じ画面で見せる（別画面に隠さない）。
  • 監査可能: 突合経路（証跡）は折り畳みで隠すが、必要なときに開いて追跡できる。
  • 静かなUI: アニメーション・効果音は最小限。スピナーとトーストのみ。
  • 1人で完結: 他者の確認・承認なしで1作業を完結できる導線。

2. ユーザーフロー / 画面遷移図

2.1 通常フロー（ハッピーパス）

  1. ユーザーがブラウザでツールを開く → SCR-001-login（Googleでログイン）
  2. 認証成功 → SCR-002-upload（3ファイルをドラッグ&ドロップ → 実行）
  3. SCR-003-processing（進行状況をポーリング表示）
  4. 処理完了 → SCR-004-result（関税率一覧・ダウンロード）
  5. 必要に応じて履歴を見る → SCR-005-history

2.2 エラーフロー

  • 許可ドメイン外でログイン試行 → SCR-006-error（「許可されたドメインのみ利用可能」）
  • 処理中に例外発生 → SCR-006-error（失敗理由を表示・履歴に戻れる）
  • 他人のジョブにアクセス → SCR-006-error（403）
  • 30日経過したジョブにアクセス → SCR-006-error（「結果は削除されました」）

5.2 タイポグラフィ

  • フォント: -apple-system, "Hiragino Kaku Gothic ProN", "Yu Gothic", Meiryo, sans-serif
  • 等幅: "Consolas", "SF Mono", "Menlo", monospace（品番・HSコード等）
  • サイズ: 本文 14px / 補助 12px / 見出しh2 18px / 見出しh1 22px
  • 行高: 1.6

5.3 スペーシング

  • 基本単位: 4px（0.25rem）。8 / 12 / 16 / 24 / 32 / 48 の倍数で構成。
  • カード内 padding: 24px
  • セクション間: 16px〜24px

5.4 角丸・影

  • 角丸: 6px（ボタン）/ 8px（カード・ドロップゾーン）
  • 影: トーストのみ box-shadow: 0 4px 12px rgba(0,0,0,.15)。他は影なし（フラットデザイン）。

5.5 アイコン

  • 絵文字を使用（CDN依存なし）。📊 ロゴ / 📜 許可証 / 📋 PO / 📦 PL / ✓ 成功 / ✗ 失敗 / ⚠ 警告 / ⬇ ダウンロード / 🔑 ログイン
  • 絵文字は色のみで意味を伝えないようテキストとセットで使用。

6. 画面外の振る舞い

6.1 トースト発火条件

イベントメッセージ色表示時間
処理完了「✓ 処理が完了しました（X品番）」緑3秒
ダウンロード「⬇ tariff_*.xlsx をダウンロードしました」緑3秒
形式エラー「PDF/Excelのみアップロード可能です」赤4秒
サイズ超過「ファイルサイズは10MB以下にしてください」赤4秒
ログアウト完了「ログアウトしました」緑2秒

6.2 モーダル発火条件

本リリースではモーダルを使用しない。確認は必要に応じてトーストとボタン状態で代替。

6.3 タイムアウト処理

  • セッション TTL: 8時間。タイムアウト後は次のリクエストで /login へリダイレクト。
  • 処理ジョブのタイムアウト: 10分（個別ジョブ）。超過時は failed に遷移しエラー表示。

6.4 ブラウザ閉じ・リロード時の挙動

  • 処理中（SCR-003）でブラウザを閉じてもジョブは継続実行される。
  • 履歴画面（SCR-005）から再開可能。
  • アップロード（SCR-002）でリロードすると選択ファイル情報は失われる（ブラウザ仕様）。
