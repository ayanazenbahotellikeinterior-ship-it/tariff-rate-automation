1. 業務ドメイン用語（貿易・輸入）

用語定義例・補足
輸入許可証 税関が発行する輸入許可通知書。商品ごとの関税率・HSコード・関税額が記載される。本ツールの主要入力。 「許可書 IS2611304.pdf」等。1案件で複数欄（最大十数欄）に商品情報が並ぶ。
関税率 輸入時に課される税率。HSコードと原産地に基づき決定される。 例: 7.4%、FREE（免税）
HSコード Harmonized System Code。世界共通の関税分類コード。日本では9〜10桁。 PL: 6208210000（10桁）／許可証: 6208.21-000（9桁、区切り付き）
税表番号 許可証PDF上のHSコード上位（4+2桁=6桁）。 例: 6208.21
品目番号 許可証PDF上のHSコード（4+2+3桁=9桁、区切り付き）。 例: 6208.21-000
欄 許可証PDF内の品目申告ブロック。< NN 欄> マーカーで区切られる。 1案件で1〜十数欄。各欄1品名と1HSコードを持つ。
S プレフィックス 許可証の関税率に付く区分コード。一般税率（Standard）と推定。 例: S 4.5%、S 9%（業務確認の余地あり）
G プレフィックス 許可証の関税率に付く区分コード。協定・特恵税率（GATT/General？）と推定。 例: G 7.4%、G FREE
FREE 免税。許可証で関税率の代わりに記載される。本ツールでは 0% に正規化。 —
原産地 許可証PDFに記載される製造国。 例: CN-CHINA
CIF価格 運賃・保険料込みの輸入価格。許可証PDFに記載。本ツールでは未使用（将来のため記録）。 —
BPR 輸入申告価格按分（割合等）に関する許可証内の項目。本ツールでは未使用。 —

2. 商品・データ構造の用語

用語定義例
品番 粗利表およびPOにおける一意な商品識別コード。本ツールの突合チェーンの起点。 ctstpjlblue、19silkpjslm1navy
Item PO上の商品ライン名または品種を示す列値。 Cotton twill stripe pajama、bottoms、dress
gender 商品の対象性別。POの列値で WOMEN / MEN 等を含む。 WOMEN、MEN
Quality 素材・生地。生地幅・目付を含むこともある。 100% cotton、100% silk、100% cotton 57/58 110GSM
素材 Quality の中核情報。突合キーとして使用。 100% cotton → トークン COTTON
Set up セット品マーカー。PL では TOPS と PANTS の2行に展開される。 PO列値 Set up → PL「WOMEN'S COTTON TWILL STRIPE PAJAMA TOPS + PANTS」
DESCRIPTION PL上の商品説明文字列。gender + 素材 + 種類を含む（素材は明記されない場合もある）。 WOMEN'S COTTON TWILL STRIPE PAJAMA TOPS
编号 PL内の通番（HLI-001 のような連番）。品番ではない。 HLI-007
PL Packing List。出荷時の梱包内容明細書。商品ごとの DESCRIPTION と HSコードが記載。 —
PO Purchase Order。発注書。品番・Item・gender・Quality 等が記載。 —
PI Proforma Invoice。仮インボイス。PO と類似の役割。 本リリースでは PO と同等に扱う
SO Sales Order。販売注文書。PO と類似の役割。 本リリースでは PO と同等に扱う
粗利表 商品ごとの粗利を管理する Google スプレッドシート。AJ列に関税数式 =PRODUCT(AA*, X%) が入る。 C列=品番、AA列=下代、AJ列=関税数式
下代 仕入価格。粗利表のAA列に入る。関税額計算のベース。 —

3. 突合・処理ロジックの用語

用語定義
三ホップ突合 「品番 → Item/gender/Quality → DESCRIPTION → HSコード → 関税率」の3段階で関税率を引くロジック。
HS正規化キー HSコードの突合用正規化形式。区切り文字を除去し、頭6桁（業務上の分類粒度）を取り出した文字列。
match_tokens PO値（gender + 素材 + Item語彙）から導出した突合用トークン。PL の DESCRIPTION に対し部分一致 AND で当てる。
部分一致 AND 突合用トークンが すべて DESCRIPTION に含まれる行をヒットとみなす方式。
語彙辞書（vocabulary） PO 側の Item / gender 語彙を PL 側の DESCRIPTION 語彙に変換するための YAML 辞書。
Set up展開 Set up マーカー付き品番について、PL の TOPS と PANTS の2行を引き、HS一致を確認する処理。
未マッチ 三ホップのいずれかのホップでマッチに失敗した状態。未マッチ理由を出力に記載。
証跡 突合経路の記録。各ホップで何にマッチしたかを保持し、監査・原因究明に使う。
正規化 表記揺れを吸収するための前処理（大文字化・空白除去・区切り除去等）。
冪等性 同じ入力に対して何回処理を実行しても、同じ出力が得られる性質。

4. UI/UX 用語

用語定義
ドロップゾーンファイルをドラッグ&ドロップで投入できる領域。クリックでファイル選択ダイアログも開く。
ステップインジケータ処理段階（アップロード → 処理中 → 結果）を可視化するヘッダ下のUI要素。
サマリカード結果画面の上部に並ぶ集計値（対象品番数・成功・未マッチ・マッチ率）の各カード。
ステータスバッジ結果テーブル各行の状態を示す小ラベル。成功（緑）/ 失敗（赤）/ 要確認（黄）。
トースト画面右下に一時的に表示される通知。3〜4秒で自動消滅。
折り畳み（details）クリックで展開・収納できる領域。HTMLネイティブの <details> を使用。
スピナー処理中を示す回転インジケータ。
プログレスバー処理進捗（0〜100%）を視覚的に示すバー。
エンプティステートデータが0件の場合の表示。「処理履歴はありません」等の説明＋次のアクションへの誘導。

5. 技術用語（Web・認証・インフラ）

用語定義
FastAPIPython の非同期 Web フレームワーク。本ツールのバックエンド層。
uvicornASGI サーバ。FastAPI を起動するランタイム。
Jinja2Python のテンプレートエンジン。サーバサイド HTML レンダリングに使用。
OAuth 2.0認可プロトコル。本ツールでは Google アカウントログインに使用（Authorization Code Flow）。
サービスアカウントGoogle Cloud の非対話的認証用アカウント。本ツールでは粗利表（Sheets）アクセスに使用。
許可ドメインOAuth ログイン時に許可するメールドメインのリスト。社外を拒否するために使う。
セッションログイン状態をサーバ・クライアント間で維持する仕組み。本ツールは暗号化Cookie。
ジョブ（Job）1案件の処理単位。アップロード→処理→結果の一連のライフサイクルを持つ。
ジョブIDジョブを一意に識別する文字列。j_YYYYMMDDHHmmssNNNN 形式。
ポーリングクライアントが定期的にサーバへ状態を問い合わせる方式。本ツールは1秒ごとに /api/jobs/{id}/status。
非同期実行HTTP リクエストの応答を待たずに別タスクで処理を進める方式。FastAPI BackgroundTasks を使用。
at-rest 暗号化ディスク・ストレージに保存されている状態での暗号化。
HTTPS通信路の暗号化（TLS）。本ツールは本番で必須。
Dockerコンテナ仮想化技術。本ツールは Dockerfile で完結させ、ホスティング先非依存。
Cloud RunGoogle Cloud のサーバレスコンテナ実行サービス。本ツールの推奨ホスティング先候補。

7. コード上の命名規則

7.1 基本ルール

  • 原則 英語スネークケース（Python の慣習）
  • 日本固有の業務用語は ローマ字スネークケース を許容（例: hinban）
  • 変数名・関数名の詳細は development-guidelines.html 4節 を参照

7.3 ファイル名・ディレクトリ名

  • Python モジュール: snake_case（permit_pdf_reader.py）
  • HTML / CSS / JS: kebab-case またはsnake_case 統一（ui-specification.html）

UI 上の表示ラベルは ui-specification.html 7節 の対応表に従う。UIラベル → 用語集 → コード命名 の3階層が常に整合していること。
