1. 設計方針

  • 単一CLIスクリプト として実装し、固定入力フォルダを起点に1コマンドで完結させる。
  • 入力種別ごとに 抽出モジュールを分離。PO は 列マッピング設定で案件ごとの様式差を吸収。
  • 抽出後は 正規化済み DataFrame に統一し、突合ロジックは純粋関数として実装。
  • 突合キーは HSコードの正規化キー（区切り除去・頭6桁）。語彙差は YAML外出しの辞書 で吸収。
  • 「Set up」マーカーは PL の TOPS+PANTS の2行に展開 し、HS一致を確認。
  • マッチ失敗は エラーにせず、未マッチ理由列に「どのホップで失敗したか」を記載 して人が補完。
  • 中間Excelは 3シート構成（結果 / 証跡 / サマリ） で監査性を確保。

3. データフロー

  1. main が config.yaml と vocabulary.yaml（語彙辞書）を読み、./inputs/ をスキャン。
  2. ファイル種別判定（拡張子 + ファイル名 + 内容ヘッダ）で許可証/PL/POに振り分け、各 reader を呼び出す。
  3. 各 reader は 正規化済み DataFrame を返す（第5節）。HSコード・関税率はこの段階で正規化キーも付与。
  4. matcher が 品番起点で三ホップ突合 し、Set up マーカーがあれば PL の TOPS+PANTS に展開・HS一致確認。
  5. excel_writer が 3シート（結果/証跡/サマリ） で中間Excelを書き出し。
  6. 処理サマリをコンソール表示 + サマリシートに記録。

6. 抽出ロジック詳細

6.1 輸入許可証 PDF（permit_pdf_reader）

  • ライブラリ: pdfplumber（テーブル抽出 + テキスト抽出の併用）。
  • 欄区切り: 正規表現 <\s*(\d+)\s*欄> で各品目ブロックを切り出し。
  • 各ブロック内で以下のラベル直後の値を取得（テキスト位置 or キーワード後の数値で判定）：
    • 品名: 英語の商品説明文（欄マーカー直下）
    • 税表番号: (\d{4}\.\d{2}(?:-\d)?)
    • 品目番号: (\d{4}\.\d{2}-\d{3})
    • 関税率: ([GS]?)\s*(\d+(?:\.\d+)?%|FREE)
  • 第一手はテーブル抽出、失敗時は正規表現フォールバック。
  • 抽出後、normalize_tariff と normalize_hs を適用。

6.2 PL Excel（pl_reader）

  • ライブラリ: openpyxl + pandas（.xls 旧形式は xlrd または事前変換）。
  • ヘッダ行・列マッピング（DESCRIPTION列・HSコード列）は config.yaml で指定。
  • HSコードの生表記をそのまま保持しつつ normalize_hs で hs_key を生成。
  • 空行・空DESCRIPTION行はスキップ。

6.3 PO（po_reader）

  • Excel優先、なければPDF。同案件で混在する場合は両方読んで連結。
  • Excel: openpyxl。 PDF: pdfplumber でテーブル抽出。
  • 列マッピング: config.yaml の po.column_mapping で「品番列名」「Item列名」「gender列名」「素材を含む列名候補リスト」「Set upマーカー列」を指定。
  • 素材抽出: Quality 列の文字列から (\d+%\s*\w+|\d+%\s*[\w-]+(?:\s+\w+)?) 等で素材部分を切り出し。
  • Set upマーカー: 指定列の値が Set up（大文字小文字無視）に一致する場合 is_setup=True。
  • match_tokens 生成: gender語（WOMEN→[WOMEN, WOMEN'S, LADIES]）+ 素材語 + 語彙辞書経由のItem語（bottoms→PANTS等）を結合。

6.4 粗利表（sheet_reader）

  • ライブラリ: google-api-python-client + google-auth。
  • 認証: サービスアカウント（閲覧権限）。粗利表側でサービスアカウントのメールに閲覧権限を共有。
  • シートID・対象シート名・データ開始行（3）・品番列（C列）は config.yaml で指定。
  • 取得結果は粗利表の行順を保持した DataFrame。

6.5 HSコード正規化（normalize_hs）

def normalize_hs(raw: str, depth: int = 6) -> str:
    """区切り文字を除去し、先頭 depth 桁を返す。

    例:
      "6208210000"  → "620821"
      "6302.31-0000" → "630231"
      "6208.21-000" → "620821"
    """
    digits = re.sub(r"\D", "", raw or "")
    return digits[:depth]

既定 depth=6（業務上のHS分類粒度）。設定で 7/9 桁にも切替可能（R-07）。

6.6 関税率正規化（normalize_tariff）

def normalize_tariff(raw: str) -> tuple[str, str]:
    """(prefix, rate) を返す。

    例:
      "S 4.5%" → ("S", "4.5%")
      "G 7.4%" → ("G", "7.4%")
      "G FREE" → ("G", "0%")
      "5.4%"   → ("", "5.4%")
    """
    m = re.match(r"\s*([GS])?\s*(FREE|\d+(?:\.\d+)?%)\s*", raw or "")
    if not m:
        return ("", "")
    prefix = m.group(1) or ""
    val = "0%" if m.group(2) == "FREE" else m.group(2)
    return (prefix, val)

7. マッチングロジック（三ホップ突合）

7.1 三ホップの定義

  1. ホップ1（PO引き当て）: profit_df.品番 → po_df.hinban を完全一致で引き、match_tokens と is_setup を取得。失敗 → 「POに品番なし」。
  2. ホップ2（PL DESCRIPTION 検索）: match_tokens を部分一致AND で pl_df.description に当て、対応行を取得。is_setup=True なら TOPS と PANTS の2行を引く。失敗 → 「PLにItem/Quality該当なし（検索キー: ...）」。Set up で2行の hs_key が不一致 → 「PLでSet up展開時にHS不一致」。
  3. ホップ3（許可証検索）: PLの hs_key で permits_df.hs_key を引き、正規化済み関税率を取得。失敗 → 「許可証に該当HSコードなし」。

7.2 部分一致 AND の評価

  • match_tokens の各トークンが すべて DESCRIPTION に部分一致する行をヒットとみなす。
  • 大文字小文字・前後空白は事前正規化。
  • 複数行ヒット時:
    • is_setup=True かつヒット2行が TOPS と PANTS → OK（HS一致確認へ）
    • それ以外で複数ヒット → 未マッチ扱い（候補を未マッチ理由に列挙、人に判断委ねる）

7.3 突合の擬似コード

for row in profit_df.itertuples():
    hinban = row.品番
    po = po_df.query("hinban == @hinban")
    if po.empty:
        result.append((None, "POに品番なし")); continue

    tokens = po.iloc[0].match_tokens
    is_setup = po.iloc[0].is_setup
    pl_hits = pl_df[pl_df.description.apply(lambda d: all(t in d for t in tokens))]

    if is_setup:
        if len(pl_hits) != 2:
            result.append((None, f"PL Set up展開で2行マッチせず（{len(pl_hits)}行）")); continue
        keys = pl_hits.hs_key.unique()
        if len(keys) > 1:
            result.append((None, f"PLでSet up展開時にHS不一致（{keys.tolist()}）")); continue
        hs_key = keys[0]
    else:
        if len(pl_hits) == 0:
            result.append((None, f"PLにItem/Quality該当なし（検索キー: {'+'.join(tokens)}）")); continue
        if len(pl_hits) > 1:
            descs = pl_hits.description.tolist()
            result.append((None, f"PLで複数候補ヒット（{descs}）人が選択")); continue
        hs_key = pl_hits.iloc[0].hs_key

    permit = permits_df.query("hs_key == @hs_key")
    if permit.empty:
        result.append((None, f"許可証に該当HSコードなし（hs_key={hs_key}）")); continue

    result.append((permit.iloc[0].関税率, None))

7.4 語彙辞書の適用（vocabulary）

PO の Item / gender の語彙を PL 側の語彙に変換するための辞書。vocabulary.yaml に外出し。

# vocabulary.yaml
gender:
  WOMEN: ["WOMEN", "WOMEN'S", "LADIES"]
  MEN: ["MEN", "MEN'S", "MENS"]
item:
  "bottoms": ["PANTS"]
  "long sleeves": ["TOPS"]
  "dress": ["DRESS"]
  "robe": ["ROBE"]
  "set up": ["SETS"]
  "Cotton twill stripe pajama": ["COTTON", "TWILL", "STRIPE", "PAJAMA"]
material:
  "100% cotton": ["COTTON"]
  "100% silk": []   # PL側にSILK表記がない案件があるため空（要 R-08 対応）
  "100% linen": ["LINEN"]

適用ロジック: PO の各値（gender / item / material）に対し、辞書のキー一致で 候補トークンのリスト を取得し、それらすべてを match_tokens に統合。

10. エラーハンドリング

事象挙動
入力フォルダが空 / 必須種別が見つからないエラーメッセージで終了（非ゼロ exit）
PDFのテキスト抽出に失敗（スキャン疑い）該当ファイル名をログ、処理は継続。サマリに記録
Excel列マッピング不一致「設定ファイル config.yaml の pl.description_column が不一致」で終了
Sheets API 認証失敗「サービスアカウント xxx@.. を粗利表に共有してください」で終了
突合の各ホップで未マッチ未マッチ理由を記載し処理継続
許可証で同一 hs_key に複数欄ヒット（要件外）warning を出して最初の値を採用。証跡列に複数欄番号を記載

11. ログ・可観測性

  • 標準出力: 処理サマリ（対象品番数 / 成功 / 未マッチ × 理由別件数 / 警告件数）。
  • ログファイル: ./logs/run_YYYYMMDD_HHmm.log。読み込んだファイル一覧、各ホップでの件数、例外スタック。
  • ログレベル: 既定 INFO、--verbose で DEBUG。

14. UI影響セクション

UI変更なし

本作業は CLI スクリプト（Pythonローカル実行）であり、ブラウザ等で操作する UI（画面）は持たない。docs/ui-specification.html への影響はない。

  • ユーザーとの接点はターミナルのコマンド実行とコンソール出力のみ。
  • 出力物は Excel ファイルだが、これはデータ成果物であり「画面」ではない。
  • Phase 2（粗利表AJ列への自動書き込み）でも引き続き UI は持たない予定。GUI化は別ステアリングで起こす。

15. テスト方針

  • ユニットテスト: normalize_hs / normalize_tariff / vocabulary / matcher / 各 reader の純粋関数部分。pytest。
  • フィクスチャ: tests/fixtures/ に実サンプルの最小セット（許可証PDF 1件・PL 1件・PO 1件）を配置。
  • 結合テスト: フィクスチャ一式で main を実行し、3シートの内容を期待値と比較。
